html的input元素的类型为image提交表单时产生的额外参数

html的input元素的类型为image提交表单时产生的额外参数

市场人员从前线发来一WiFi热点的抓包文件，让我们做登录过程分析。
同事分析后发现，登录过程中post的一个输入参数找不到。分析了用登录页面，登录页面里的js脚本，外部引用的js脚本。使用没有发现参数在那里来的。

我拿到抓包数据文件，把登录页面数据拷贝出来，保存为本地html页面，用浏览器打开，启动开发者工具，进行分析，也没特别发现。
把form的数据提交方法从POST改为GET。点击登录按钮，页面跳转后，在地址栏中清晰的能看到提交的数据中确实有form表单里不存在的出现。反复尝试几次，发现参数名不变，但参数的值在不断变化。看来这个参数值是客户端生成的，和服务端没关系。

form表单里面有一个input 变量为confirm，提交的参数里没有名字为confirm的变量，多出来的两个变量为confirm.x和confirm.y。x, y难不成是一个坐标。

id为confirm的input 元素的类型为image。

以前还没用过这种类型。查了下资料，发现传递的多余参数实际是点击图片提交时鼠标在图片上的坐标。

如果想屏蔽参数提交，把name的值设为”none”即可。

顺便查下input 元素支持的类型，还真多。

值 描述
button 定义可点击的按钮（通常与 JavaScript 一起使用来启动脚本）。
checkbox 定义复选框。
color New 定义拾色器。
date New 定义 date 控件（包括年、月、日，不包括时间）。
datetime New 定义 date 和 time 控件（包括年、月、日、时、分、秒、几分之一秒，基于 UTC 时区）。
datetime-local New 定义 date 和 time 控件（包括年、月、日、时、分、秒、几分之一秒，不带时区）。
email New 定义用于 e-mail 地址的字段。
file 定义文件选择字段和 “浏览…” 按钮，供文件上传。
hidden 定义隐藏输入字段。
image 定义图像作为提交按钮。
month New 定义 month 和 year 控件（不带时区）。
number New 定义用于输入数字的字段。
password 定义密码字段（字段中的字符会被遮蔽）。
radio 定义单选按钮。
range New 定义用于精确值不重要的输入数字的控件（比如 slider 控件）。
reset 定义重置按钮（重置所有的表单值为默认值）。
search New 定义用于输入搜索字符串的文本字段。
submit 定义提交按钮。
telNew 定义用于输入电话号码的字段。
text 默认。 定义一个单行的文本字段（默认宽度为 20 个字符）。
time New 定义用于输入时间的控件（不带时区）。
url New 定义用于输入 URL 的字段。
week New 定义 week 和 year 控件（不带时区）。

new 的那些是html 5增加的。
