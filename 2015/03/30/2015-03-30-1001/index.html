项目管理

  • 什么是管理
    1. 管理是为了实现组织未来目标的活动
    2. 管理的工作本质是协调
    3. 管理的工作存在于组织中
    4. 管理的工作重点是对人进行管理(管的住，理的顺)
  • 项目管理的四个层次
    1. 把复杂的事情简单化——WBS 分解
    2. 把简单的事情量化——-时间\成本\资源\质量
    3. 把量化的东西模版化——统一的标准、指标、要求和做法
    4. 把模版的事情信息化——组织过程资产库、经验教训库。
  • 项目工作流程
    1. 启动
    2. 规划
    3. 执行
    4. 监控(跟踪、审查和调整项目的进展和绩效)
    5. 收尾
  • 管理纬度
    1. 范围管理——用结构华方法把复杂事情简单化
    2. 时间管理——向最长路径抢时间，向最短路径抢资源
    3. 成本管理——追求资源的优化配置
    4. 质量管理
  • 价值观
    1. 交付成功的产品与团队
    2. 必须获得N多人的认可
  • 时间管理
    1. 如何通过别人完成工作
    2. 沟通或者去沟通的路上

  • 项目管理者的角色

    1. 项目管理者(Project)
      a. 理解项目价值，管理项目目标
      b. 获得预期的商业回报等结果

      用正确的方法做正确的事。

    2. 团队建设者(Team)
      a. 带出有战斗力的团队成员
      b. 交出”5+5>10”的团队

      打造团队的灵魂

    3. 组织能力提升者(Organization)
      a. 协调建立不依赖于个人的组织
      b. 促进流程、方法与技术的持续改进

      基于流程持续的改进和积累

    4. 问题解决者(Solution)
      a. 解决最困难的项目问题
      b. 沟通协调解决项目干系人的冲突

      基于全局，达到满意

  • 核心能力

    1. 挖掘业务价值 —— 解决了什么问题，增加了什么价值
    2. 理解客户需求 —— 区分期望和需求，清晰准确的达成共识
    3. 把握团队目标 —— 有限信息下的决策, 目标分解与沟通
    4. 帮助团队寻找工作方法 —— 启发、辅导；承担必要的风险

    5. 维持秩序 —— 优秀的管理制度都是简单的。

      重要性从上而下，越靠上，重要性越高。

  • 项目管理的技能要求

    1. 优秀的领导能力
    2. 高校的时间管理能力
    3. 快速的应变能力
    4. 良好的人际交往能力
    5. 非凡的沟通能力
    6. 高校的激励能力
    7. 运用项目管理知识的能力
  • 团队协作的五大障碍
    1. 无视结果
    2. 惧怕冲突
    3. 欠缺投入
    4. 逃避责任
    5. 缺乏信任
  • 打造团队
    1. 公布工作目标
    2. 基于集体的成就进行奖励
    3. 强调注重集体成就
    4. 克服自我主义
    5. 建立客观的态度
    6. 奖励那些真正为集体利益做出贡献的成员
